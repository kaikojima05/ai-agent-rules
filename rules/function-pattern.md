# 関数の編集ルール

## 切り出し基準
1. if や for が 2 層以上ネストする場合は、ローカルに関数として処理を切り出す
2. 条件式内に 3 つ以上の条件がある場合は、条件式自体を変数として切り出すこと
3. 複数ファイルに跨って同一の処理がある場合は、編集対象のファイルと同階層に `utils.ts` を作成し、ヘルパー関数を追加すること

## コメント
1. ヘルパー関数の作成時は、関数の先頭に処理内容をコメントで記述する
   - Why: 実装を読まなくても意図が伝わるようにし、再利用判断のコストを下げるため
```typescript
// ダメな例
// details 中から必要なデータを filter() して sort() する関数

// 良い例
// 明細データの中から割引データを除き、金額の高い順に並び替える関数
```

## 型
1. 戻り値の型は極力推論させる
   - ただし、戻り値の型が分岐する場合は、明示的に戻り値の型を定義する

## 配置
1. 新しく定義する関数やコンポーネントはファイルの最終行に追加する

## スタイル
1. 可読性を優先し、宣言的な書き方を選ぶこと
  - 例: **reduce()** ではなく **filter().map()** を使う
  - ただし、パフォーマンスが計測上ボトルネックになっている箇所には適用しない
