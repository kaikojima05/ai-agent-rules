## 目的

書きっぱなしになりがちな規約を、エージェントの裁量任せにせず自動で検証する。
AGENTS.md と `.claude/rules/**/*.md` の規約と、現在の差分を突き合わせて違反箇所を列挙する役割。

## 実行フロー

### Step 1: スコープを引数から判定

引数で diff のスコープを切り替える。

| 引数 | 対象 | コマンド |
|---|---|---|
| 無し | ステージング済み | `git diff --staged` |
| `all` | ステージング + 未ステージング | `git diff HEAD` |
| `branch` | ベースブランチからの全差分 | `git diff $(git merge-base HEAD master)..HEAD` |

ベースブランチが `master` でない場合は `git symbolic-ref refs/remotes/origin/HEAD` から推測する。

### Step 2: 規約を読み込み、構造化する

`.claude/rules/**/*.md` の構成はリポジトリごとに変わるため **ハードコードしない**。
毎回 Glob で拾い直すこと。

1. `Glob` で `.claude/rules/**/*.md` と `AGENTS.md` を全部拾う
2. それぞれを `Read` で読み、項目を以下のカテゴリに分類する
  - **禁止事項** — 「〜してはならない」「禁止する」を含む箇条書き
  - **必須事項** — 「必ず〜すること」「〜すること」を含む箇条書き
  - **推奨事項** — 「〜が望ましい」「推奨」を含む箇条書き
3. 各項目を `{rule, category, source_file, source_line}` の構造で保持する

### Step 3: 差分を抽出する

Step 1 のコマンドで diff を取り、`+` 行（追加された行）と `-` 行（削除された行）を抽出する。
ファイルパス・行番号・変更内容を保持しておくこと。

### Step 4: 違反検出

各規約項目に対して、差分行を照合する。検出手段は 2 種類に切り分ける。

- **機械的に判定できるもの** — `Grep` / 正規表現で十分
  - 例: `from "dayjs"` の import、`reduce(` の使用、`../../` の相対 import
- **読解が必要なもの** — エージェントが LLM として判定
  - 例: 「マジックナンバーの使用禁止」「30 行以上の追加禁止」「変数名で意図を伝えない」

照合結果は以下の形式で出力する。

```
[<file>:<line>] <違反した規約>
  → 該当コード: <該当行>
  → 修正案: <具体的な書き換え案>
  → 出典: <rule ファイル>:<行>
```

### Step 5: 後続スキルへの誘導

違反一覧を提示したあと、ユーザーに次のアクションを尋ねる。

- 全件まとめて直すなら `/clean-code` を提案
- 個別に修正するなら該当ファイルだけを Edit する旨を提案

ユーザー承認なしに勝手に修正してはならない。あくまで **検出 + 提案** までで停止する。

## 注意事項

- 差分が大きいときは `git diff --stat` で件数を先に出し、ユーザーに続行確認を取る
- AGENTS.md の `!only-codex` のように、特定エージェント限定の規約は除外する判断を入れる
- 規約が曖昧で違反判定に迷うものは「グレー」として別カテゴリで列挙し、勝手に違反扱いしない
- 出力は違反件数 0 件でも「0 件でした」と必ず報告する
