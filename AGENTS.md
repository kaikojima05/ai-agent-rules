# 規約

## 命名規則
1. 変数名・関数名・クラス名は既存の実装パターンに合わせる
2. 関数の追加・rename 時は命名案を複数提案し、ユーザーの許可を得る

## 禁止事項
1. マジックナンバーの使用
2. dayjs を `"dayjs"` から呼び出すこと
  - 必ず `"@/modules/dayjs"` に用意された関数を呼び出すこと
  - ただし、`"@/modules/dayjs"` の dayjs の使用は非推奨としているので、その他の適切な関数を使用する

## 出力制限
1. 一度に10行以上のコード追加・変更は禁止（テストコードは除く）
2. コードの追加・変更時は必ず意図をユーザーに説明する（テストコードは除く）
  - 変数名や関数名で意図を簡略化して伝えることは禁止、必ず本質的な説明を行うこと

## !only-codex
1. HTTP リクエストの送信はサンドボックス外で行うこと
