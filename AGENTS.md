# 規約

## 設計原則
1. SOLID 原則を遵守すること

## 命名規則
1. 変数名・関数名・クラス名は既存の実装パターンに合わせる

## 禁止事項
1. マジックナンバーの使用
2. 2階層以上深いディレクトリから import する時に `../` で繋げて書くこと
  - 例） `../../~` ではなく `@front/services/~` と書くこと
  - ※ 1階層までであれば `../` と書くことは許可する

## 出力制限
1. 一度に30行以上のコード追加・変更は禁止（テストコードは除く）
2. コードの追加・変更時は必ず意図をユーザーに説明する（テストコードは除く）
  - 変数名や関数名で意図を簡略化して伝えることは禁止、必ず本質的な説明を行うこと

## HTTP Request
1. HTTP リクエストの送信はサンドボックス外で行うこと
